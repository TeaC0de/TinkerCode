Building My Blog: HTML Basics in Action

Introduction

When I decided to start my blog, I realized it was the perfect opportunity to learn HTML from scratch. With MDN Web Docs as my guide, I dove into the world of web development and discovered just how much thought goes into structuring a simple web page.

What I Learned

  • Semantic Elements: I learned how <header>, <main>, <section>, and <footer> create meaningful divisions on a webpage, improving accessibility and readability. 
    <header>
    <h1>Welcome to My Blog</h1>
</header>
<main>
    <section>
        <h2>About Me</h2>
        <p>This is where I share my journey of learning web development.</p>
    </section>
</main>
<footer>
    <p>© 2024 My Blog</p>
</footer>
  • Headings and Text: Structuring content with <h1> through <h6> for headings and <p> for paragraphs gave my blog clarity and hierarchy. 
    <h1>Main Title</h1>
<h2>Subheading</h2>
<p>This is a paragraph explaining the details of the topic.</p>
  • Links and Navigation: Using <a> tags, I created hyperlinks that connected different sections of my blog and external resources. 
    <a href="https://developer.mozilla.org/en-US/docs/Web/HTML">Learn more about HTML</a>
  • Images and Metadata: Adding visuals with <img> tags and understanding <meta> tags to improve SEO were fun and practical discoveries. 
    <meta name="description" content="A blog about learning HTML basics and building projects.">
<img src="my-image.jpg" alt="An image representing my blog theme">

Challenges and Solutions

Learning HTML was not without its hurdles. Here are some examples of challenges I faced and how I solved them:

  • Challenge 1: Proper Nesting of Elements

    One challenge I encountered was understanding how to properly nest elements. It was easy to accidentally break the structure by placing elements in the wrong order.

    <div>
            <p>This is a paragraph inside a div.</p>
            <span>This is a span inside the div.</span>
        </div>

    Solution: I learned that elements should be nested correctly to avoid display issues. Practice and referring to MDN documentation helped me grasp this concept quickly.

  • Challenge 2: Ensuring Proper Link Structure

    Creating links that worked correctly was initially tricky. I needed to make sure that relative and absolute paths were used appropriately to avoid broken links.

    <a href="about.html">About Me</a> 
        <a href="https://www.example.com">Visit Example</a>

    Solution: By experimenting with different types of paths and understanding their usage, I became more comfortable with creating links that lead to the right places.

  • Challenge 3: Understanding Attribute Values

    Another challenge was understanding the proper use of attribute values, such as alt text for images and target for links.

    <img src="image.jpg" alt="A scenic view of mountains">
        <a href="https://example.com" target="_blank">External Link</a>

    Solution: I researched best practices and learned that using descriptive alt text improves accessibility, and the target="_blank" attribute opens links in a new tab when needed.
